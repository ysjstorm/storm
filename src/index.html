SCSS [scss]

page > reset page/reset.scss
page > customer [고객] page/customer.scss
page > marketing [마케팅] page/marketing.scss

- Guide -

  • 작성 방법은 Nesting 방식으로 한다.
  • scss 파일 최상단에 charset을 반드시 선언한다.
  • 주석은 css에 컴파일 되지 않는 한줄 주석을 지향한다.
  • 초기 sass 작업 방식이 익숙하지 않아 실수가 잦을 수도 있고 이 실수를 모르고 넘어갈 수 있으니 각 작업자의 세밀하고 꼼꼼한 작업을 요한다. (scss의 컴파일 후 css 수시 확인도 필요하다.)
  • common/reset.scss - tag에 대한 기본 style을 선언하며, 초기에 작업자들의 논의를 통해 update 할 수 있도록 한다. (수정 및 추가는 반드시 협의 후에 진행 하도록 한다.)
  • common/layout.scss - layout의 style을 선언하며 페이지 전역에 사용되는 UIO 또한 추가 할 수 있다.
  • common/common.scss - 페이지 전역까지는 아니지만 여러 페이지에서 사용되는 공통 UIO의 내용을 담는다. 추후 UIO 양에 따라 성향/용도에 맞게 분리한다.
  • mixins/mixins.scss - mixin의 내용을 담으며 추후 페이지가 많아 질 경우 용도에 맞게 확장할 수 있다. mixin이 사용되는 scss에 import 한다. (기본적으로 모든 scss에 import 하여도 무방하다.)
  • page/각 페이지에 맞는 scss - 각 scss 상황에 맞게 import와 image path를 선언한다.

CSS [css] - 절대경로 : http://img1.tmon.kr/resources/m/css/

customer [고객센터] customer.css
marketing [마케팅] marketing.css

- Guide -

  • 실제 html 파일에 link되는(External) css는 reset.css와 각 페이지 용도에 맞는 css 만을 link 한다.
    ex) 고객센터의 페이지에는 customer.css 만 link가 된다. (reset.css는 scss에서 import 한다.)
    ex) reset.css의 용도는 윈도우팝업 페이지에 link 하고, 윈도우팝업의 컨텐츠 style은 head 태그안에 Internal stlyle로 작업한다. (layout이 고유한 단발 페이지에 대해서도 상황에 따라 reset.css 를 사용하게 될 것으로 예상된다.)
  • 실제 link 하는 css라 할지라도 css의 작업은 없다. 모든 style 작업은 scss에서 작업되며 css는 scss를 컴파일하여 link로 사용될 뿐이다.
